## 1단계 — GitHub 저장소 만들기 (10분, Chrome 가능)

- https://github.com 으로 이동.

### 1-2. 저장소 생성

  - Visibility: **Public**
  - **Add a README file** 체크 해제 (이미 README 있음)
  - .gitignore, license: 추가하지 않음
- **Create repository** 클릭.

### 1-3. 결과 확인

## 2단계 — Oracle Cloud 가입 + ARM VM 생성 (25분, Chrome 가능)

### 2-1. 무료 계정 가입
- https://www.oracle.com/cloud/free/ 접속.
- **Start for free** 클릭.
- 입력값:
  - 국가: **Canada**
  - 이름·이메일: 사용자에게 확인
  - 지역(Home Region): **Canada Southeast (Toronto)** 선택 (변경 불가, 신중)
- 본인 인증: 휴대폰 SMS 인증 → 사용자에게 코드 요청.
- 신용카드 정보 입력: 사용자에게 입력 요청. (과금되지 않음을 안내)
- 가입 완료까지 5~15분 소요. 인증 메일 도착 대기.

### 2-2. ARM VM 인스턴스 생성
- https://cloud.oracle.com 로그인 후, 왼쪽 햄버거 메뉴 → **Compute** → **Instances**.
- **Create instance** 클릭.
- 입력값:
  - Name: `church-booking-server`
  - Image and shape:
    - Image: **Ubuntu 22.04**
    - Shape: **Change shape** 클릭 → **Ampere** 탭 → **VM.Standard.A1.Flex** 선택
    - OCPU: **4**, Memory: **24 GB**
  - Networking: 기본값 사용 (자동으로 VCN 생성됨)
  - SSH keys: **Generate a key pair for me** 선택
    - **Save Private Key** 와 **Save Public Key** 둘 다 다운로드. 사용자에게 두 파일을 안전한 곳에 저장하라고 안내.
- **Create** 클릭. Provisioning 완료까지 1~2분.

### 2-3. 방화벽 규칙 추가
- 인스턴스 상세 페이지 → **Virtual cloud network** 링크 클릭.
- **Security Lists** → **Default Security List** 클릭.
- **Add Ingress Rules** 클릭. 다음 두 규칙 추가:
  1. Source CIDR `0.0.0.0/0`, Destination Port `80` (HTTP)
  2. Source CIDR `0.0.0.0/0`, Destination Port `443` (HTTPS)

### 2-4. 결과 확인
- 인스턴스 페이지에서 **Public IP Address**를 복사해 사용자에게 보고.
- 다음 메시지 전달:
  > "서버가 준비됐습니다. 공인 IP는 `<IP>` 입니다.
  > 다운로드한 SSH 개인키 파일 경로를 알려주세요. 다음 단계에서 서버 접속에 사용합니다."

## 3단계 — DNS 설정 (15분, Chrome 가능)

### 3-1. kwanglim.ca 도메인 관리 콘솔 접속
- 사용자에게 도메인이 어디서 관리되는지 확인 (예: Cloudflare, GoDaddy, Namecheap).
- 해당 사이트로 이동 후 사용자 로그인 요청.

### 3-2. A 레코드 추가
- DNS 관리 페이지에서 **Add record** 또는 유사 버튼 클릭.
- 입력값:
  - Type: **A**
  - Name/Host: `booking`
  - Value/Points to: 2단계에서 받은 Oracle 서버 공인 IP
  - TTL: 기본값 (자동 또는 3600)
  - Proxy(Cloudflare인 경우): **DNS only** (회색 구름)
- 저장.

### 3-3. 전파 확인
- 사용자에게 안내: "DNS 전파에 5~30분 소요됩니다. 그동안 4단계로 넘어가도 됩니다."
- 확인 명령(사용자가 터미널에서): `dig booking.kwanglim.ca +short`

## 4단계 — 서버 세팅 (1~1.5시간) ⚠️ Chrome으로는 불가

이 단계는 SSH 접속이 필요하므로 **Chrome 에이전트로는 실행하지 못합니다.**
사용자에게 "이 부분은 Cowork(데스크톱)나 터미널에서 진행해야 합니다"라고 안내한 뒤,
다음 체크리스트를 사용자에게 그대로 전달하세요. 실행은 사용자/Cowork이 담당합니다.

```
□ SSH 접속: ssh -i <개인키경로> ubuntu@<공인IP>
□ 시스템 업데이트: sudo apt update && sudo apt upgrade -y
□ Node.js 22 LTS 설치 (NodeSource):
    curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
    sudo apt install -y nodejs
□ Nginx 설치: sudo apt install -y nginx
□ PM2 전역 설치: sudo npm install -g pm2
□ Git clone:
    cd ~ && git clone https://github.com/<username>/church-booking-system.git
    cd church-booking-system
□ 의존성 설치:
    cd server && npm install
    cd ../client && npm install && npm run build
□ .env 파일 생성 (server/.env)
    SMTP_USER=kmcreservation@gmail.com
    SMTP_PASS=<앱비밀번호>
    ADMIN_EMAIL=kmcreservation@gmail.com
    NOTIFICATION_EMAIL=canadakc@gmail.com
    BASE_URL=https://booking.kwanglim.ca
    GOOGLE_CLIENT_ID=<발급된 Client ID>
    PORT=3001
□ DB 초기화: cd server && node db/init.js && node db/migrate.js
□ PM2로 백엔드 실행:
    pm2 start index.js --name booking-api
    pm2 save && pm2 startup  (출력된 명령 복사 실행)
□ Nginx 설정 (/etc/nginx/sites-available/booking):
    server {
      listen 80;
      server_name booking.kwanglim.ca;
      location / { root /home/ubuntu/church-booking-system/client/dist; try_files $uri /index.html; }
      location /api/ { proxy_pass http://localhost:3001; proxy_set_header Host $host; }
    }
    sudo ln -s /etc/nginx/sites-available/booking /etc/nginx/sites-enabled/
    sudo nginx -t && sudo systemctl reload nginx
□ Let'\''s Encrypt SSL:
    sudo apt install -y certbot python3-certbot-nginx
    sudo certbot --nginx -d booking.kwanglim.ca
□ 방화벽 (서버 안): sudo ufw allow 'Nginx Full' && sudo ufw allow OpenSSH && sudo ufw enable
```

사용자에게 "각 단계 끝날 때마다 결과를 알려주세요"라고 요청.

## 5단계 — Google OAuth 콜백 URL 추가 (5분, Chrome 가능)

### 5-1. Google Cloud Console 접속
- https://console.cloud.google.com 접속.
- 프로젝트 선택: `kwanglim-space`.
- 왼쪽 메뉴 → **APIs & Services** → **Credentials**.

### 5-2. OAuth Client ID 편집
- 기존 Client ID 클릭.
- **Authorized JavaScript origins**에 다음 추가:
  - `https://booking.kwanglim.ca`
- **Save** 클릭.

### 5-3. (선택) OAuth 동의 화면 게시
- **OAuth consent screen** → **Publish app** 클릭 (테스트 → 프로덕션).
- 사용자에게 "이 단계는 선택사항입니다. 지금은 건너뛰어도 됩니다"라고 안내해도 됨.

## 트러블슈팅 메모

- **DNS 전파 안 됨**: 30분 대기. `dig booking.kwanglim.ca` 로 IP 확인.
- **SSL 발급 실패**: 80번 포트가 열려 있고 DNS가 전파됐는지 먼저 확인.
- **이메일 안 옴**: `.env`의 `SMTP_PASS`가 Gmail 앱 비밀번호(공백 포함된 16자)인지 확인.
- **Google 로그인 실패**: Authorized JavaScript origins에 `https://booking.kwanglim.ca` 가 정확히 등록됐는지.
- **캘린더 안 뜸**: `cd server && node db/init.js` 실행 빠뜨렸는지 확인.
